What it is

stryke-mssql is a Microsoft SQL Server / Azure SQL client for stryke: parametrized query and execute, transaction batches, scalar/exists helpers, and schema introspection — over tiberius, the pure-Rust TDS driver. No FreeTDS, no ODBC. The cdylib owns one tokio runtime and presents a blocking facade, caching a client per (host, db, auth, encrypt).

Quick start

use Mssql

var %conn = ( host => "localhost", database => "app",
              username => "sa", password => $ENV{MSSQL_PASS}, trust_cert => 1 )

val @users = Mssql::query("SELECT id, name FROM users WHERE active = @P1", params => [1], %conn)
p Mssql::scalar("SELECT COUNT(*) FROM users", %conn)

Mssql::execute("UPDATE users SET name = @P1 WHERE id = @P2", params => ["Ada", 42], %conn)

Mssql::batch(
    ["INSERT INTO audit (msg) VALUES ('x')", "UPDATE users SET seen = 1 WHERE id = 42"],
    %conn,
)

Typed rows

Each cell is converted to JSON by trying the TDS types in order (int, float, bool, string, decimal, uuid, datetime, binary), so arbitrary result sets round-trip without a per-query schema. Decimals keep full precision as strings, binary is base64, and datetimes are ISO-8601.
